# MwjRunner 使用手册 — 进阶篇

> 本篇为进阶内容，涵盖 HTML 报告、退出码、脱敏、数据驱动、并发、hooks、协议扩展、质量门禁、平台等高级功能。基础入门请参阅 [使用手册-入门篇](使用手册-入门篇.md)。

## 16. HTML 报告

MwjRunner 支持生成单文件 HTML 报告，适合在浏览器中查看测试结果。

### 16.1 生成 HTML 报告

在 `--report` 参数中加入 `html`：

```bash
uv run mwjrunner run examples/cases/login_profile.yaml --base-url http://127.0.0.1:8000 --report console,json,html
```

### 16.2 报告路径

```text
reports/{run_id}/report.html
```

### 16.3 HTML 报告包含内容

- 运行概览：run_id、开始/结束时间、总耗时。
- 统计卡片：用例、步骤、断言、错误的通过/失败/错误计数。
- 用例列表：可折叠展开，失败或错误用例默认展开。
- 步骤明细：请求方法/URL/Headers/Body、响应状态码/Headers/Body。
- 断言表格：类型、路径、期望值、实际值、消息。
- 提取变量表格：变量名、路径、值。
- 错误信息。
- 脱敏后的敏感字段（password、token、authorization 等显示为 `***REDACTED***`）。

### 16.4 安全说明

HTML 报告采用 template.html + JSON 注入方案。JSON 数据中的 `</script>` 已转义，不存在 HTML 注入风险。报告为单文件，CSS 和 JS 内联，无需联网，可离线打开。

直接在浏览器打开模板文件 `mwjrunner/reports/template.html` 可预览 mock 演示数据，方便前端调试。

## 17. 退出码说明

MwjRunner 的 CLI 应返回稳定退出码。

| 退出码 | 含义 | 说明 |
| --- | --- | --- |
| 0 | 全部通过 | 所有用例通过，或只有跳过用例 |
| 1 | 断言失败 | 程序正常运行，但接口结果不符合预期 |
| 2 | 用例或配置错误 | YAML 写错、字段缺失、配置错误等 |
| 3 | 引擎内部错误或能力尚未实现 | MwjRunner 自身出现未预期错误，或当前命令暂未支持 |

CI 可以根据退出码判断流水线是否通过。

## 18. 敏感信息脱敏说明

### 18.1 什么是敏感信息

敏感信息是不能随便出现在日志和报告里的内容，例如：

- password
- token
- access_token
- refresh_token
- cookie
- set-cookie
- secret
- authorization

### 18.2 为什么要脱敏

如果这些信息明文出现在报告里，别人拿到报告后可能可以：

- 冒充用户登录。
- 访问内部接口。
- 获取隐私数据。
- 操作测试环境或生产环境数据。

所以 MwjRunner 必须在输出报告前统一脱敏。

### 18.3 脱敏示例

原始请求：

```json
{
  "username": "demo",
  "password": "123456"
}
```

报告中应该显示：

```json
{
  "username": "demo",
  "password": "***REDACTED***"
}
```

原始 Header：

```http
Authorization: Bearer demo-token
```

报告中应该显示：

```http
Authorization: ***REDACTED***
```

## 19. pytest 使用边界

MwjRunner 的核心执行框架和核心断言机制不能使用 pytest。

也就是说：

- 用户写的接口测试用例不能变成 pytest 测试函数。
- MwjRunner 不能依赖 pytest 来发现和执行接口用例。
- MwjRunner 不能依赖 pytest assertion rewriting 来实现断言。
- MwjRunner 的报告不能依赖 Allure。

允许的情况：

- 项目自身开发时，可以使用 pytest 测试 MwjRunner 的 Python 代码。
- `examples/api/` FastAPI 示例服务也可以使用 pytest 做自身接口测试。

简单理解：

> pytest 可以是开发者检查代码质量的工具，但不能成为 MwjRunner 产品能力的一部分。

## 20. 常见问题

### 20.1 `uv` 命令不存在

现象：

```text
uv: command not found
```

原因：

本机没有安装 UV，或者 UV 不在 PATH 中。

处理：

- 先安装 UV。
- 重新打开终端。
- 再执行：

```bash
uv --version
```

### 20.2 Python 版本太低

现象：

```text
Python 3.10.x
```

原因：

当前项目要求 Python `>=3.13`。

处理：

- 安装 Python 3.13 或更高版本。
- 确认终端中的 `python` 指向新版本。

### 20.3 FastAPI 服务没有启动

现象：

```text
Connection refused
```

原因：

MwjRunner 请求 `http://127.0.0.1:8000`，但 FastAPI 服务没启动。

处理：

进入 `examples/api`，启动服务：

```bash
uv run uvicorn app.main:app --reload
```

### 20.4 端口 8000 被占用

现象：

```text
Address already in use
```

原因：

本机已有其他程序占用了 8000 端口。

处理方式一：关闭占用 8000 的程序。

处理方式二：换一个端口，例如 8001：

```bash
uv run uvicorn app.main:app --reload --port 8001
```

然后运行用例时也要改 base-url：

```bash
uv run mwjrunner run examples/cases/health.yaml --base-url http://127.0.0.1:8001 --report console,json
```

### 20.5 用例文件路径写错

现象：

```text
Case file not found: examples/cases/health.yaml
```

原因：

用例文件不存在，或者当前目录不对。

处理：

1. 确认你在 MwjRunner 项目根目录。
2. 确认文件路径存在。
3. 使用正确路径重新运行。

### 20.6 YAML 缩进错误

现象：

```text
YAML parse error
```

原因：

YAML 对缩进很敏感，不能随便多空格或少空格。

错误示例：

```yaml
steps:
- name: 服务健康状态
 request:
   method: GET
```

正确示例：

```yaml
steps:
  - name: 服务健康状态
    request:
      method: GET
```

处理：

- 使用两个空格缩进。
- 不要混用 Tab 和空格。
- 对照本文档示例修改。

### 20.7 断言失败怎么看

如果看到：

```text
expected: down
actual: ok
```

意思是：

- 用例期望值是 `down`。
- 接口实际返回的是 `ok`。
- 所以断言失败。

处理：

- 如果接口返回是正确的，就修改用例期望值。
- 如果用例期望是正确的，就检查接口实现。

### 20.8 token 没有提取到

可能原因：

1. 登录接口失败了。
2. JSONPath 写错了。
3. 响应结构和预期不一致。

示例：

如果响应是：

```json
{
  "data": {
    "token": "demo-token"
  }
}
```

提取路径应该是：

```yaml
extract:
  token: $.data.token
```

如果写成：

```yaml
extract:
  token: $.token
```

就会提取失败。

### 20.9 报告里为什么看不到完整 token

这是正常现象。

MwjRunner 会对 token、password、authorization 等敏感字段脱敏，避免泄露。

如果报告中看到：

```text
token: ***REDACTED***
```

说明脱敏功能正在生效。

## 21. 推荐学习顺序

如果你是第一次使用，建议按下面顺序学习：

1. 先理解 `/health` 健康检查接口。
2. 学会运行健康检查用例。
3. 学会看 Console 报告。
4. 学会看 JSON 报告。
5. 学习登录接口。
6. 学习 token 提取。
7. 学习后续步骤复用 token。
8. 学习断言失败怎么定位。
9. 学习敏感信息为什么要脱敏。
10. 再学习 tags、retry、fail-fast、HTML 报告等增强能力。

## 22. M3 新增功能速查

### 22.1 目录批量执行

```bash
uv run mwjrunner run examples/cases/ --base-url http://127.0.0.1:8000 --report console,json
```

递归发现目录下所有 `*.yaml` / `*.yml` 文件并执行。以 `_` 开头的文件会被跳过。

### 22.2 环境配置

在项目中创建 `envs/test.yaml`：

```yaml
base_url: http://test-api.example.com
timeout: 30
variables:
  admin_user: test_admin
```

使用环境配置运行：

```bash
uv run mwjrunner run cases/ --env test --report console
```

### 22.3 数据驱动

在用例中使用 inline data：

```yaml
name: 登录参数化
data:
  - username: admin
    password: admin123
  - username: guest
    password: guest456
steps:
  - name: 登录
    request:
      method: POST
      url: /api/login
      json:
        username: ${username}
        password: ${password}
    assertions:
      - type: status_code
        expected: 200
```

也可以引用外部 CSV/JSON 文件：

```yaml
name: 批量测试
data_file: data/users.csv
steps: ...
```

### 22.4 失败重试

```bash
uv run mwjrunner run cases/ --base-url http://127.0.0.1:8000 --retry 2
```

失败的 case 最多重试 2 次（共执行 3 次）。也可以在 YAML 中为单个 case 指定 `retry: N`。

### 22.5 fail-fast

```bash
uv run mwjrunner run cases/ --base-url http://127.0.0.1:8000 --fail-fast
```

遇到第一个失败 case 后停止执行，后续 case 标记为 skipped。

### 22.6 标签和优先级过滤

```bash
# 只执行 smoke 标签的用例
uv run mwjrunner run cases/ --base-url http://127.0.0.1:8000 --tags smoke

# 排除 wip 标签
uv run mwjrunner run cases/ --base-url http://127.0.0.1:8000 --exclude-tags wip

# 只执行 P0 和 P1 优先级
uv run mwjrunner run cases/ --base-url http://127.0.0.1:8000 --priority P0,P1
```

### 22.7 并发执行

```bash
uv run mwjrunner run cases/ --base-url http://127.0.0.1:8000 --workers 4
```

使用 4 个线程并发执行 case，每个 case 变量上下文独立隔离。

### 22.8 用例校验

```bash
uv run mwjrunner validate cases/
```

只校验 YAML 格式是否正确，不发送请求。

### 22.9 项目初始化

```bash
uv run mwjrunner init
```

在当前目录创建推荐的项目结构（cases/、envs/、reports/、mwjrunner.yaml）。

### 22.10 Hooks 生命周期

在用例中声明 hook：

```yaml
name: 需要认证的用例
hooks:
  before_case: myproject.hooks.setup_auth
  after_case: myproject.hooks.cleanup
  before_step: myproject.hooks.log_step
  after_step: myproject.hooks.check_step
steps: ...
```

支持的 hook 类型：

| Hook | 触发时机 |
|------|----------|
| `before_case` | 用例执行前 |
| `after_case` | 用例执行后 |
| `before_step` | 每个步骤执行前 |
| `after_step` | 每个步骤执行后 |

hook 函数接收变量上下文 dict，可以读写变量。

## 22.11 YAML 用例完整字段参考

以下是 MwjRunner YAML 用例支持的所有字段，供进阶用户查阅。

### 用例顶层字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 用例名称 |
| `steps` | list | 是 | 步骤列表 |
| `tags` | list[string] | 否 | 标签，用于过滤 |
| `priority` | string | 否 | 优先级（P0/P1/P2/P3） |
| `variables` | dict | 否 | 用例级变量 |
| `data` | list[dict] | 否 | 内联数据驱动 |
| `data_file` | string | 否 | 外部数据文件路径（CSV/JSON） |
| `retry` | int | 否 | 失败重试次数 |
| `hooks` | dict | 否 | 生命周期钩子 |
| `auth` | dict | 否 | 认证配置 |

### step 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 步骤名称 |
| `request` | dict | 是 | 请求配置 |
| `assertions` | list | 否 | 断言列表 |
| `extract` | dict | 否 | 变量提取 |
| `variables` | dict | 否 | 步骤级变量（覆盖用例级） |

### request 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `method` | string | 是 | HTTP 方法（GET/POST/PUT/DELETE 等） |
| `url` | string | 是 | 请求路径（会拼接 base_url） |
| `headers` | dict | 否 | 请求头 |
| `query` | dict | 否 | URL 查询参数 |
| `cookies` | dict | 否 | 请求 Cookie |
| `json` | any | 否 | JSON 请求体 |
| `data` | any | 否 | 表单数据 |
| `body` | string | 否 | 原始请求体 |
| `timeout` | number | 否 | 请求超时（秒） |
| `files` | list | 否 | 文件上传（`[{field: "file", path: "./a.txt"}]`） |

### assertion 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 断言类型 |
| `expected` | any | 是 | 期望值 |
| `path` | string | 否 | JSONPath 路径（json_path 类型必填） |
| `target` | string | 否 | 目标字段名（header/cookie 类型使用） |
| `mode` | string | 否 | `soft`（默认，继续执行）或 `hard`（失败立即停止） |

**支持的断言类型：**

| type | 说明 | 必填字段 |
|------|------|----------|
| `status_code` | HTTP 状态码 | expected |
| `json_path` | JSONPath 取值比较 | path, expected |
| `body_contains` | 响应体包含字符串 | expected |
| `contains` | body_contains 的别名 | expected |
| `response_time` | 响应时间（ms）不超过 | expected |
| `json_schema` | JSON Schema 校验 | expected（schema dict） |
| `regex` | 正则匹配响应体 | expected（正则表达式） |
| `header` | 响应头值比较 | target, expected |
| `cookie` | 响应 Cookie 值比较 | target, expected |

### extract 字段

简写形式（默认 json_path）：

```yaml
extract:
  token: $.data.token
```

完整形式：

```yaml
extract:
  token:
    type: json_path
    path: $.data.token
    optional: false
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 否 | 提取类型（默认 `json_path`） |
| `path` | string | 是 | 提取路径 |
| `optional` | bool | 否 | 提取失败是否忽略（默认 false） |

**支持的提取类型：** `json_path`、`header`、`cookie`、`regex`

### 变量函数

在变量值中使用 `${__函数名()}` 调用内置函数：

| 函数 | 说明 | 示例 |
|------|------|------|
| `${__timestamp()}` | 当前时间戳（秒） | `1715234567` |
| `${__uuid()}` | UUID v4 | `a1b2c3d4-...` |
| `${__random_phone()}` | 随机手机号 | `13812345678` |
| `${__random_int()}` | 随机整数（0-9999） | `4231` |
| `${__random_int(1,100)}` | 指定范围随机整数 | `42` |
| `${__random_str()}` | 随机字符串（8位） | `aB3kLm9x` |
| `${__random_str(16)}` | 指定长度随机字符串 | `aB3kLm9xPq2wYz7n` |
| `${__md5(hello)}` | MD5 哈希 | `5d41402abc4b2a76...` |

## 23. 导入外部用例

MwjRunner 支持从 Postman Collection 和 OpenAPI/Swagger 文件导入用例。

### 23.1 导入 Postman Collection

```bash
uv run mwjrunner import postman_collection.json --format postman --output cases/imported
```

支持 Postman Collection v2.0 和 v2.1 格式。导入后会在 `cases/imported/` 目录生成对应的 YAML 用例文件。

### 23.2 导入 OpenAPI / Swagger

```bash
uv run mwjrunner import swagger.json --format openapi --output cases/generated
```

支持 OpenAPI 3.x 和 Swagger 2.0。自动解析 path/query/header 参数、request body schema，并猜测成功状态码生成断言。

### 23.3 导入后的用例结构

导入生成的 YAML 用例与手写用例格式完全一致，可以直接执行或手动修改完善。

```bash
# 导入后直接执行
uv run mwjrunner run cases/imported/ --base-url http://127.0.0.1:8000 --report console
```

## 24. 通知配置

执行完成后可以自动发送通知到钉钉、企业微信或邮件。

### 24.1 在项目配置中声明通知

在 `mwjrunner.yaml` 中添加 `notifications` 字段：

```yaml
notifications:
  - type: dingtalk
    webhook_url: https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN
  - type: wecom
    webhook_url: https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY
  - type: email
    smtp_host: smtp.example.com
    smtp_port: 465
    smtp_user: test@example.com
    smtp_password: ${EMAIL_PASSWORD}
    recipients:
      - team@example.com
    subject: "MwjRunner 执行报告"
```

支持的通知类型：

| type | 必填字段 | 说明 |
|------|----------|------|
| `dingtalk` | `webhook_url` | 钉钉机器人 Webhook |
| `wecom` | `webhook_url` | 企业微信机器人 Webhook |
| `email` | `smtp_host`, `smtp_user`, `smtp_password`, `recipients` | SMTP 邮件 |

### 24.2 通知内容

通知会包含执行摘要：用例总数、通过数、失败数、通过率、耗时。失败时会附带失败用例名称。

## 25. 认证方案

### 25.1 Bearer / Basic 认证

在环境配置中声明认证方案，引擎自动注入 Authorization header：

```yaml
# envs/dev.yaml
name: dev
base_url: http://127.0.0.1:8000
auth:
  type: bearer
  token: ${API_TOKEN}
```

```yaml
# Basic 认证
auth:
  type: basic
  username: admin
  password: ${ADMIN_PASSWORD}
```

### 25.2 OAuth2 认证

支持 `client_credentials` 和 `password` 两种 grant type：

```yaml
auth:
  type: oauth2
  grant_type: client_credentials
  token_url: https://auth.example.com/oauth/token
  client_id: ${CLIENT_ID}
  client_secret: ${CLIENT_SECRET}
```

引擎会自动向 `token_url` 发起请求获取 access_token，并注入到后续请求的 Authorization header 中。

## 26. 协议扩展

MwjRunner 通过协议适配器支持 WebSocket 和 gRPC 协议。协议适配器已实现，平台的场景编排功能可以调用这些协议。

> 注意：当前 CLI 的 `mwjrunner run` 命令仅支持 HTTP 协议。WebSocket 和 gRPC 用例需要通过平台的场景编排功能执行，或通过 Python API 编程调用。后续版本将支持在 YAML 用例中直接声明 `protocol` 字段。

### 26.1 WebSocket

WebSocket 适配器支持连接、发送消息和接收响应。

**通过平台场景编排使用：**

在平台"场景编排"中添加 WebSocket 步骤时，配置以下参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| target | WebSocket 地址 | `ws://127.0.0.1:8000/ws` |
| action | 操作类型 | `send_recv` 或 `send` |
| payload | 发送的消息内容 | `{"type": "ping"}` |
| timeout | 超时时间（秒） | `10` |

支持的 action：
- `send_recv`：发送消息并等待响应
- `send`：只发送不等待响应

**通过 Python API 调用：**

```python
from mwjrunner.protocols.websocket import WebSocketAdapter
from mwjrunner.protocols import ProtocolRequest

adapter = WebSocketAdapter()
result = adapter.execute(ProtocolRequest(
    protocol="websocket",
    action="send_recv",
    target="ws://127.0.0.1:8000/ws",
    payload={"type": "ping"},
    options={"timeout": 5},
))

if result.is_success:
    print(result.response.payload)  # 响应内容（自动 JSON 解析）
```

**依赖安装：**

```bash
uv add websockets
```

### 26.2 gRPC

gRPC 适配器支持 Unary RPC 和 Server Streaming，支持动态 Server Reflection。

**通过平台场景编排使用：**

| 参数 | 说明 | 示例 |
|------|------|------|
| target | 格式：`host:port/package.Service/Method` | `localhost:50051/helloworld.Greeter/SayHello` |
| payload | 请求消息体（JSON 格式） | `{"name": "MwjRunner"}` |
| headers | 元数据键值对 | `{"authorization": "Bearer xxx"}` |
| options.tls | 是否使用 TLS | `false` |
| options.timeout | 超时时间（秒） | `30` |
| options.streaming | 是否 Server Streaming | `false` |
| options.proto_file | .proto 文件路径（可选） | `protos/hello.proto` |

**通过 Python API 调用：**

```python
from mwjrunner.protocols.grpc import GrpcAdapter
from mwjrunner.protocols import ProtocolRequest

adapter = GrpcAdapter()
result = adapter.execute(ProtocolRequest(
    protocol="grpc",
    action="call",
    target="localhost:50051/helloworld.Greeter/SayHello",
    payload={"name": "MwjRunner"},
    headers={"authorization": "Bearer token123"},
    options={"tls": False, "timeout": 10},
))

if result.is_success:
    print(result.response.payload)  # {"message": "Hello MwjRunner"}
```

**gRPC 特性：**
- Unary RPC 调用
- Server Streaming RPC（返回消息列表）
- 动态 Server Reflection（无需预编译 stub）
- TLS / 非 TLS 连接
- Metadata 传递
- 超时控制

**依赖安装：**

gRPC 支持需要额外安装依赖（可选）：

```bash
uv add grpcio grpcio-reflection protobuf
```

## 27. 质量门禁

质量门禁用于在 CI 中自动判断测试是否达标。

### 27.1 配置质量门禁

在 `mwjrunner.yaml` 中配置：

```yaml
quality_gate:
  max_failure_rate: 0.1         # 失败率不超过 10%
  max_error_rate: 0.05          # 错误率不超过 5%
  max_response_time_ms: 3000    # 最大响应时间不超过 3000ms
```

### 27.2 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 全部通过 |
| 1 | 存在断言失败 |
| 2 | 用例加载或配置错误 |
| 3 | 执行引擎内部错误 |
| 4 | 质量门禁不通过 |

当质量门禁不通过时，退出码为 4，CI 流水线会自动标记为失败。

## 28. 报告插件

如果内置的 Console / JSON / HTML 报告不满足需求，可以注册自定义报告插件。

### 28.1 编写自定义报告器

```python
from pathlib import Path
from mwjrunner.reports.plugin import ReporterPlugin
from mwjrunner.reports.model import RunResult

class MyReporter(ReporterPlugin):
    @property
    def name(self) -> str:
        return "my_report"

    def render(self, result: RunResult) -> str:
        # result 是 RunResult 对象
        return f"通过率: {result.summary.passed_cases}/{result.summary.total_cases}"

    def write(self, result: RunResult, output_path: Path) -> None:
        content = self.render(result)
        output_path.write_text(content)
```

### 28.2 注册插件

在项目配置中注册：

```yaml
plugins:
  reporters:
    - myproject.reporters.MyReporter
```

## 29. 数据源扩展

除了内联 `data` 和外部 CSV/JSON 文件，MwjRunner 还提供了 Excel 数据加载函数。

### 29.1 CSV 数据驱动

```yaml
name: CSV 数据驱动
data_file: data/users.csv
steps:
  - name: 创建用户
    request:
      method: POST
      url: /users
      json:
        username: ${username}
        email: ${email}
    assertions:
      - type: status_code
        expected: 201
```

CSV 文件第一行为列名（即变量名），后续每行为一组测试数据。

### 29.2 JSON 数据驱动

```yaml
name: JSON 数据驱动
data_file: data/users.json
steps:
  - name: 创建用户
    request:
      method: POST
      url: /users
      json:
        username: ${username}
        email: ${email}
    assertions:
      - type: status_code
        expected: 201
```

JSON 文件内容为对象数组：`[{"username": "a", "email": "a@x.com"}, ...]`

### 29.3 Excel 数据加载

Excel 数据加载函数已实现（`mwjrunner.data.load_excel_data`），可在 hook 中使用：

```python
# myproject/hooks.py
from mwjrunner.data import load_excel_data

def load_test_data(ctx: dict):
    """在 before_case hook 中加载 Excel 数据。"""
    ctx["test_data"] = load_excel_data("data/users.xlsx", sheet="Sheet1")
```

> 注意：当前 `data_file` 字段仅支持 `.csv` 和 `.json` 格式。Excel 文件需要通过 hook 函数加载。后续版本将在 `data_file` 中直接支持 `.xlsx`。

**依赖安装：**

```bash
uv add openpyxl
```

### 30.5 场景编排

## 31. 常见问题

### Q: 执行用例时报 502 错误

Windows 下 httpx 可能自动读取系统代理。MwjRunner 已内置绕过系统代理的处理，如果仍有问题，检查环境变量中是否设置了 `HTTP_PROXY` 或 `HTTPS_PROXY`。

### Q: 变量提取失败，提示响应体为空

检查上一步请求是否成功返回了数据。提取失败时错误信息会包含 status_code 和响应体预览，方便定位问题。

### Q: gRPC 用例报错找不到 grpcio

gRPC 是可选依赖，需要手动安装：

```bash
uv add grpcio grpcio-reflection protobuf
```

### Q: WebSocket 报错找不到 websockets

WebSocket 是可选依赖，需要手动安装：

```bash
uv add websockets
```

### Q: 平台登录后页面空白

检查前端是否正确构建（`npm run build`），Nginx 是否配置了 `try_files $uri $uri/ /index.html`。

### Q: WebSocket 连接失败

确认 Nginx 配置了 WebSocket 代理（需要 `proxy_http_version 1.1` 和 `Upgrade` 头），详见部署手册第 3.3 节。
