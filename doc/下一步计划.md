# MwjRunner 下一步计划

本文档用于管理 MwjRunner 后续开发任务。每次开发前必须先阅读本文档；如果用户提出的新任务不在本文档中，必须先更新本文档，再开始开发。

## 使用规则

1. 每次开发前先确认“当前任务”。
2. 每个任务必须包含目标、范围、输出物和验收标准。
3. 任务执行过程中如果范围变化，先更新本文档，再继续执行。
4. 任务完成后，将状态更新为“已完成”，并补充实际结果。
5. 后续代码实现必须优先服务于本文档中的任务，不做与当前任务无关的扩展。
6. 当 CLI、用例格式、示例接口、报告格式或核心约束变化时，必须同步更新 `doc/使用手册.md`、`doc/需求规格说明书.md` 和 `doc/技术方案.md`。
7. 开发过程中发现但未在当前任务内立即修复的 P0-P3 问题，必须记录到 `doc/开发问题跟踪.md`。

### T18：失败重试（retry）

状态：已完成

优先级：P1

目标：

范围：

- CLI 新增 `--retry` 参数，默认 0（不重试）。

- 每次 attempt 独立记录 request、response、assertions、errors。
- 最终 case 状态以最后一次 attempt 为准。
- 报告中展示 attempt 次数和每次结果。
- 重试之间无额外等待（首期不实现 backoff）。

输出物：

- retry 调度逻辑。
- attempt 结果模型扩展。
- 覆盖重试成功、重试仍失败、case 级覆盖的单元测试。
- 同步更新使用手册。

验收标准：

- `--retry 2` 时，失败 case 最多执行 3 次（1 + 2 重试）。
- 报告中可看到每次 attempt 的详细结果。
- 重试后成功的 case 最终状态为 passed。
- case 级 `retry` 字段可覆盖 CLI 全局设置。

输出物：

- 覆盖 fail-fast 触发、与 retry 配合、全部通过不触发的单元测试。
- 同步更新使用手册。

验收标准：

- 报告中被跳过的 case 标记为 skipped。
