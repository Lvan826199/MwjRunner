# MwjRunner 下一步计划

本文档用于管理 MwjRunner 后续开发任务。每次开发前必须先阅读本文档；如果用户提出的新任务不在本文档中，必须先更新本文档，再开始开发。

## 使用规则

1. 每次开发前先确认“当前任务”。
2. 每个任务必须包含目标、范围、输出物和验收标准。
3. 任务执行过程中如果范围变化，先更新本文档，再继续执行。
4. 任务完成后，将状态更新为“已完成”，并补充实际结果。
5. 后续代码实现必须优先服务于本文档中的任务，不做与当前任务无关的扩展。
6. 当 CLI、用例格式、示例接口、报告格式或核心约束变化时，必须同步更新 `doc/使用手册.md`、`doc/需求规格说明书.md` 和 `doc/技术方案.md`。

M1 最小引擎阶段。

### T9：实现 Console 和 JSON 报告

优先级：P1

输出终端摘要和结构化 JSON 报告。

实际结果：

- 运行成功时退出码为 0。

### T8：实现变量替换和 JSONPath 提取

状态：已完成

优先级：P1

支持登录后提取 token，并在后续 step 中通过 `${token}` 复用。

- 复用自研 JSONPath 解析逻辑，避免新增三方依赖。
- 补充变量替换和 JSONPath 提取单元测试。

验收标准：

- 登录接口返回 token 后，profile 请求能使用该 token。
- token 在报告中脱敏。

### T7：实现自研基础断言

状态：已完成

优先级：P0

目标：

实现 MwjRunner 自研断言引擎，支持 status_code、json_path、contains 等基础断言类型。

实际结果：

- 未使用 pytest 作为核心断言机制；pytest 仅用于项目自身单元测试。
- 补充自研断言模块单元测试。

验收标准：

- 能对 HTTP 响应执行 status_code 断言。
- 能对 JSON 响应执行 json_path 断言。
- 断言失败时返回清晰的错误信息。
