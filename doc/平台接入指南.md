# 平台接入指南

本文档面向测试平台开发者，说明如何将 MwjRunner 执行引擎集成到测试平台中。

## 接入架构概览

```
┌─────────────────────────────────────────────────┐
│                  测试平台                         │
│                                                  │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐   │
│  │ 用例管理  │───▶│ 调度中心  │───▶│ 报告展示  │   │
│  └──────────┘    └──────────┘    └──────────┘   │
│                       │                          │
└───────────────────────┼──────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│              MwjRunner 执行引擎                   │
│                                                  │
│  输入: YAML 用例文件 + CLI 参数                    │
│  输出: JSON 报告 + HTML 报告 + 退出码             │
│                                                  │
└─────────────────────────────────────────────────┘
```

平台通过 **子进程调用 CLI** 的方式与 MwjRunner 交互，引擎以 YAML 文件为输入、JSON 报告为输出，平台解析 JSON 获取结构化结果。

---

## 1. 调用方式

### 1.1 CLI 命令格式

```bash
mwjrunner run <用例文件路径> [选项]
```

### 1.2 核心参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `<path>` | 用例文件路径（必填） | `cases/login.yaml` |
| `--base-url` | HTTP 请求基础地址 | `--base-url http://api.test.com` |
| `--report` | 报告类型，逗号分隔 | `--report json,html,console` |
| `--report-dir` | 报告输出目录 | `--report-dir /tmp/reports` |
| `--var KEY=VALUE` | 运行时变量，可重复 | `--var token=abc --var env=staging` |

### 1.3 平台调用示例（Python subprocess）

```python
import subprocess
import json
from pathlib import Path

def run_case(case_file: str, base_url: str, variables: dict = None, report_dir: str = None):
    """调用 MwjRunner 执行单个用例文件。"""
    cmd = ["mwjrunner", "run", case_file, "--base-url", base_url, "--report", "json,html"]

    if report_dir:
        cmd.extend(["--report-dir", report_dir])

    for key, value in (variables or {}).items():
        cmd.extend(["--var", f"{key}={value}"])

    result = subprocess.run(cmd, capture_output=True, text=True, timeout=300)

    return {
        "exit_code": result.returncode,
        "stdout": result.stdout,
        "stderr": result.stderr,
    }
```

## 2. 退出码含义

引擎通过进程退出码传递执行状态，平台必须根据退出码判断执行结果：

| 退出码 | 含义 | 平台处理建议 |
|--------|------|-------------|
| `0` | 全部通过 | 标记用例为 PASSED |
| `1` | 存在断言失败 | 标记用例为 FAILED，解析报告展示失败详情 |
| `2` | 存在执行错误（网络、配置、变量等） | 标记用例为 ERROR，提示用户排查 |
| `3` | 引擎内部错误 | 标记为系统异常，通知平台运维 |

## 3. JSON 报告结构

报告文件位于 `<report-dir>/<run_id>/result.json`，`run_id` 格式为 `YYYYMMDD-HHMMSS-<8位hex>`。

### 3.1 顶层结构

```json
{
  "run_id": "20260512-090929-8e2e7767",
  "started_at": "2026-05-12T08:09:29.000000+00:00",
  "ended_at": "2026-05-12T08:09:30.000000+00:00",
  "summary": { ... },
  "cases": [ ... ],
  "errors": [ "运行级错误信息" ]
}
```

### 3.2 summary 对象

```json
{
  "total_cases": 1,
  "passed_cases": 1,
  "failed_cases": 0,
  "error_cases": 0,
  "total_steps": 2,
  "passed_steps": 2,
  "failed_steps": 0,
  "error_steps": 0,
  "total_assertions": 6,
  "failed_assertions": 0,
  "total_errors": 0,
  "elapsed_ms": 616.1
}
```

平台可直接使用 summary 字段生成统计面板和趋势图表。

### 3.3 cases 数组

```json
{
  "name": "登录后获取用户信息",
  "source_file": "cases/login_profile.yaml",
  "status": "passed",
  "steps": [ ... ],
  "errors": []
}
```

`status` 取值：`passed` | `failed` | `error`

### 3.4 steps 数组

```json
{
  "name": "登录成功",
  "status": "passed",
  "elapsed_ms": 320.5,
  "request": {
    "method": "POST",
    "url": "http://api.example.com/api/login",
    "headers": { "Content-Type": "application/json" },
    "query": {},
    "cookies": {},
    "body": "{\"username\": \"demo\", \"password\": \"***REDACTED***\"}",
    "timeout": 30.0
  },
  "response": {
    "status_code": 200,
    "headers": { "content-type": "application/json" },
    "cookies": {},
    "body": "{\"code\": 0, \"message\": \"success\", \"data\": {\"token\": \"***REDACTED***\"}}",
    "elapsed_ms": 318.2
  },
  "assertions": [ ... ],
  "extracts": [ ... ],
  "errors": []
}
```

### 3.5 assertions 数组

```json
{
  "type": "status_code",
  "passed": true,
  "expected": 200,
  "actual": 200,
  "path": null,
  "target": null,
  "mode": "soft",
  "message": "status_code 断言通过"
}
```

断言类型（`type`）：`status_code` | `json_path` | `body_contains`

### 3.6 extracts 数组

```json
{
  "name": "token",
  "type": "json_path",
  "path": "$.data.token",
  "value": "***REDACTED***",
  "extracted": true,
  "optional": false,
  "message": "变量提取成功"
}
```

## 4. 用例文件格式（YAML）

平台需要将用户在界面上编排的用例转换为以下 YAML 格式：

```yaml
name: 用例名称
tags: [smoke, auth]
variables:
  username: demo
  password: "123456"
steps:
  - name: 步骤名称
    request:
      method: POST
      url: /api/login
      headers:
        Content-Type: application/json
      json:
        username: ${username}
        password: ${password}
    extract:
      token: $.data.token
    assertions:
      - type: status_code
        expected: 200
      - type: json_path
        path: $.code
        expected: 0
      - type: body_contains
        expected: success

  - name: 后续步骤（可引用上一步提取的变量）
    request:
      method: GET
      url: /api/profile
      headers:
        Authorization: Bearer ${token}
    assertions:
      - type: status_code
        expected: 200
```

### 4.1 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 用例名称 |
| `tags` | list[string] | 否 | 标签，用于筛选 |
| `variables` | dict | 否 | 用例级变量 |
| `steps` | list | 是 | 步骤列表 |
| `steps[].name` | string | 是 | 步骤名称 |
| `steps[].request.method` | string | 是 | HTTP 方法 |
| `steps[].request.url` | string | 是 | 请求路径（相对或绝对） |
| `steps[].request.headers` | dict | 否 | 请求头 |
| `steps[].request.json` | any | 否 | JSON 请求体 |
| `steps[].request.data` | any | 否 | Form 请求体 |
| `steps[].request.body` | string | 否 | 原始请求体 |
| `steps[].request.query` | dict | 否 | URL 查询参数 |
| `steps[].request.cookies` | dict | 否 | Cookies |
| `steps[].request.timeout` | float | 否 | 超时秒数，默认 30 |
| `steps[].extract` | dict | 否 | 变量提取，key=变量名, value=JSONPath |
| `steps[].assertions` | list | 否 | 断言列表 |

### 4.2 变量系统

- 变量通过 `${变量名}` 语法引用
- 变量来源优先级：CLI `--var` > 用例 `variables` 字段
- 步骤间变量传递：前一步 `extract` 提取的变量可在后续步骤中使用
- 变量可用于 url、headers、json、data、body 等所有请求字段

### 4.3 断言类型

| type | 说明 | 必填字段 |
|------|------|---------|
| `status_code` | HTTP 状态码断言 | `expected` |
| `json_path` | JSONPath 取值断言 | `path`, `expected` |
| `body_contains` | 响应体包含子串 | `expected` |

## 5. 敏感信息脱敏

引擎会自动对以下字段进行脱敏处理，平台无需额外处理：

- 请求/响应中的 `token`、`password`、`secret`、`authorization`、`cookie` 等敏感字段
- 脱敏后显示为 `***REDACTED***`
- 脱敏作用于 JSON 报告和 HTML 报告

平台在展示报告时可直接使用脱敏后的数据，无需二次处理。

## 6. 报告文件结构

每次执行生成独立目录：

```
<report-dir>/
└── <run_id>/
    ├── result.json    # 结构化 JSON 报告（平台解析用）
    ├── report.html    # 单文件 HTML 报告（可直接提供下载或内嵌展示）
    └── run.log        # 执行日志
```

- `result.json`：平台解析的核心数据源
- `report.html`：单文件、无外部依赖、可离线打开，适合作为附件下载或 iframe 嵌入
- `run.log`：执行过程日志，排查问题时使用

## 7. 平台接入典型流程

```
1. 用户在平台 UI 编排用例
       │
       ▼
2. 平台将用例转换为 YAML 文件写入临时目录
       │
       ▼
3. 平台调用 subprocess 执行:
   mwjrunner run /tmp/cases/xxx.yaml \
     --base-url http://target-api \
     --report json,html \
     --report-dir /tmp/reports \
     --var env=staging
       │
       ▼
4. 等待进程结束，获取退出码
       │
       ▼
5. 解析 <report-dir>/<run_id>/result.json
       │
       ▼
6. 将结果写入平台数据库，更新用例执行状态
       │
       ▼
7. 可选：存储 report.html 供用户下载查看
```

## 8. 平台适配注意事项

### 8.1 run_id 获取

`run_id` 由引擎自动生成（格式 `YYYYMMDD-HHMMSS-<hex8>`），平台需要在 `report-dir` 下扫描最新目录获取：

```python
from pathlib import Path
import json

def get_latest_report(report_dir: str) -> dict:
    """获取最新一次执行的 JSON 报告。"""
    report_path = Path(report_dir)
    run_dirs = sorted(report_path.iterdir(), reverse=True)
    if not run_dirs:
        return None
    result_file = run_dirs[0] / "result.json"
    if result_file.exists():
        return json.loads(result_file.read_text(encoding="utf-8"))
    return None
```

或者更可靠的方式：指定唯一的 `report-dir` 给每次执行，执行后直接遍历该目录下唯一的子目录。

### 8.2 超时控制

平台应在 subprocess 层设置超时，防止用例执行无限挂起：

```python
try:
    result = subprocess.run(cmd, capture_output=True, text=True, timeout=600)
except subprocess.TimeoutExpired:
    # 标记为超时，清理进程
    pass
```

### 8.3 并发执行

当前引擎为单进程单用例执行模式。平台如需并发，应在平台调度层启动多个子进程，每个进程执行不同用例文件，使用不同的 `--report-dir` 避免冲突。

### 8.4 环境隔离

- 每次执行使用独立的 `--report-dir`，避免报告覆盖
- 通过 `--base-url` 切换目标环境（开发/测试/预发/生产）
- 通过 `--var` 注入环境相关变量（如 token、账号等）

### 8.5 用例文件管理

平台可选择以下方式管理用例文件：

1. **动态生成**：用户在 UI 编排后，平台实时生成 YAML 写入临时目录
2. **Git 仓库**：用例存储在 Git 仓库，平台 clone 后执行
3. **对象存储**：用例存储在 OSS/S3，执行前下载到本地

### 8.6 HTML 报告嵌入

HTML 报告为单文件、无外部依赖，可通过以下方式在平台中展示：

```html
<!-- iframe 嵌入 -->
<iframe src="/api/reports/{run_id}/report.html" width="100%" height="800px"></iframe>

<!-- 或提供下载链接 -->
<a href="/api/reports/{run_id}/report.html" download>下载 HTML 报告</a>
```

## 9. 依赖与部署

### 9.1 环境要求

- Python >= 3.13
- 依赖管理：UV（推荐）或 pip

### 9.2 安装

```bash
# 使用 UV（推荐）
uv pip install -e /path/to/mwjrunner

# 或使用 pip
pip install -e /path/to/mwjrunner
```

安装后 `mwjrunner` 命令即可在 PATH 中使用。

### 9.3 Docker 部署建议

```dockerfile
FROM python:3.13-slim
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv
WORKDIR /app
COPY . .
RUN uv sync
ENTRYPOINT ["uv", "run", "mwjrunner"]
```

平台可将引擎打包为 Docker 镜像，通过容器调度执行用例。

## 10. 完整接入示例（Python）

```python
"""测试平台接入 MwjRunner 的完整示例。"""

import json
import subprocess
import tempfile
from pathlib import Path

import yaml


def create_case_file(case_data: dict, work_dir: Path) -> Path:
    """将平台用例数据转换为 YAML 文件。"""
    case_file = work_dir / "case.yaml"
    case_file.write_text(yaml.dump(case_data, allow_unicode=True), encoding="utf-8")
    return case_file


def execute_case(case_file: Path, base_url: str, variables: dict = None) -> dict:
    """执行用例并返回结构化结果。"""
    report_dir = case_file.parent / "reports"
    cmd = [
        "mwjrunner", "run", str(case_file),
        "--base-url", base_url,
        "--report", "json,html",
        "--report-dir", str(report_dir),
    ]
    for key, value in (variables or {}).items():
        cmd.extend(["--var", f"{key}={value}"])

    proc = subprocess.run(cmd, capture_output=True, text=True, timeout=600)

    # 查找生成的报告
    report_dirs = sorted(report_dir.iterdir()) if report_dir.exists() else []
    result_data = None
    if report_dirs:
        result_file = report_dirs[0] / "result.json"
        if result_file.exists():
            result_data = json.loads(result_file.read_text(encoding="utf-8"))

    return {
        "exit_code": proc.returncode,
        "report": result_data,
        "html_path": str(report_dirs[0] / "report.html") if report_dirs else None,
    }


# 使用示例
if __name__ == "__main__":
    case = {
        "name": "健康检查",
        "steps": [{
            "name": "GET /health",
            "request": {"method": "GET", "url": "/health"},
            "assertions": [{"type": "status_code", "expected": 200}],
        }],
    }

    with tempfile.TemporaryDirectory() as tmp:
        work_dir = Path(tmp)
        case_file = create_case_file(case, work_dir)
        result = execute_case(case_file, base_url="http://localhost:8000")

        if result["exit_code"] == 0:
            print("PASSED")
        else:
            print(f"FAILED (exit_code={result['exit_code']})")
            if result["report"]:
                print(json.dumps(result["report"]["summary"], indent=2, ensure_ascii=False))
```
