# MwjRunner 部署手册

本文档说明如何在生产环境中部署 MwjRunner 引擎和 Web 管理平台。

---

## 1. 环境要求

| 组件 | 最低要求 |
|------|----------|
| 操作系统 | Linux (Ubuntu 20.04+) / Windows Server 2019+ / macOS 12+ |
| Python | >= 3.13 |
| Node.js | >= 18（仅前端构建需要） |
| 包管理 | UV（Python）、npm/pnpm（前端） |
| 数据库 | SQLite（默认）/ PostgreSQL（可选，生产推荐） |
| 内存 | >= 2GB（引擎）/ >= 4GB（平台全栈） |
| 磁盘 | >= 10GB（含报告存储） |

---

## 2. 引擎部署（MwjRunner CLI）

### 2.1 安装

```bash
# 克隆项目
git clone https://gitee.com/xiaozai-van-liu/MwjRunner.git
cd MwjRunner

# 安装依赖
uv sync

# 验证安装
uv run mwjrunner --help
```

### 2.2 配置环境文件

```bash
# 创建环境配置
mkdir -p envs
cat > envs/prod.yaml << 'EOF'
name: production
base_url: https://api.your-domain.com
auth:
  type: bearer
  token: ${PROD_API_TOKEN}
variables:
  timeout: 30
EOF
```

### 2.3 执行用例

```bash
# 单文件执行
uv run mwjrunner run cases/smoke.yaml --env prod --report json,html

# 目录批量执行
uv run mwjrunner run cases/ --env prod --tags smoke --workers 4 --report json,html

# 指定报告输出目录
uv run mwjrunner run cases/ --env prod --report-dir /data/reports
```

### 2.4 定时任务（Linux cron）

```bash
# 每天凌晨 2 点执行回归测试
0 2 * * * cd /opt/MwjRunner && uv run mwjrunner run cases/ --env prod --tags regression --report json,html --report-dir /data/reports 2>&1 >> /var/log/mwjrunner.log
```

## 3. 平台部署（Web 管理界面）

### 3.1 后端部署

```bash
cd platform/backend

# 安装依赖
uv sync

# 启动（开发模式）
uv run uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

# 启动（生产模式，使用 gunicorn + uvicorn worker）
uv run gunicorn app.main:app \
  -w 4 \
  -k uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8080 \
  --timeout 120 \
  --access-logfile /var/log/mwjrunner-api.log
```

**环境变量：**

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `DATABASE_URL` | 数据库连接串 | `sqlite+aiosqlite:///data/mwjrunner.db` |
| `CORS_ORIGINS` | 允许的跨域来源 | `*` |
| `SECRET_KEY` | Token 签名密钥 | 随机生成 |

### 3.2 前端构建与部署

```bash
cd platform/frontend

# 安装依赖
npm install

# 构建生产包
npm run build

# 产物在 dist/ 目录，部署到 Nginx/CDN
```

### 3.3 Nginx 配置

```nginx
server {
    listen 80;
    server_name mwjrunner.your-domain.com;

    # 前端静态文件
    location / {
        root /opt/MwjRunner/platform/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;
    }
}
```

### 3.4 Systemd 服务（Linux）

```ini
# /etc/systemd/system/mwjrunner-api.service
[Unit]
Description=MwjRunner API Server
After=network.target

[Service]
Type=simple
User=mwjrunner
WorkingDirectory=/opt/MwjRunner/platform/backend
ExecStart=/opt/MwjRunner/platform/backend/.venv/bin/gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8080
Restart=always
RestartSec=5
Environment=DATABASE_URL=sqlite+aiosqlite:///opt/MwjRunner/platform/backend/data/mwjrunner.db

[Install]
WantedBy=multi-user.target
```

```bash
# 启用并启动服务
sudo systemctl daemon-reload
sudo systemctl enable mwjrunner-api
sudo systemctl start mwjrunner-api
sudo systemctl status mwjrunner-api
```

## 4. Docker 部署（推荐）

### 4.1 Dockerfile（后端）

```dockerfile
FROM python:3.13-slim

WORKDIR /app

# 安装 UV
RUN pip install uv

# 复制项目
COPY platform/backend/ .

# 安装依赖
RUN uv sync --frozen

# 创建数据目录
RUN mkdir -p data storage

EXPOSE 8080

CMD ["uv", "run", "uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8080"]
```

### 4.2 Dockerfile（前端）

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY platform/frontend/package*.json ./
RUN npm ci
COPY platform/frontend/ .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY deploy/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### 4.3 Docker Compose

```yaml
version: '3.8'

services:
  api:
    build:
      context: .
      dockerfile: deploy/Dockerfile.backend
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data
      - ./storage:/app/storage
    environment:
      - DATABASE_URL=sqlite+aiosqlite:///app/data/mwjrunner.db
    restart: always

  web:
    build:
      context: .
      dockerfile: deploy/Dockerfile.frontend
    ports:
      - "80:80"
    depends_on:
      - api
    restart: always
```

```bash
# 一键启动
docker-compose up -d

# 查看日志
docker-compose logs -f api

# 停止
docker-compose down
```

## 5. 数据库迁移

当前使用 SQLAlchemy 自动建表（`Base.metadata.create_all`），首次启动自动创建所有表。

如需手动迁移或升级：

```bash
cd platform/backend

# 查看当前表结构
uv run python -c "
from app.models import Base
for table in Base.metadata.sorted_tables:
    print(f'{table.name}: {[c.name for c in table.columns]}')
"
```

**生产环境建议：**
- 定期备份 SQLite 数据库文件（`data/mwjrunner.db`）
- 大规模使用时迁移到 PostgreSQL
- 使用 Alembic 管理数据库版本迁移

## 6. 初始化管理员

首次部署后，创建管理员账号：

```bash
cd platform/backend

uv run python -c "
import asyncio
from app.core.database import async_session_factory, init_db
from app.models.user import User, Team
from app.api.users import hash_password

async def setup():
    await init_db()
    async with async_session_factory() as db:
        # 创建默认团队
        team = Team(name='默认团队', description='系统默认团队')
        db.add(team)
        await db.flush()

        # 创建管理员
        admin = User(
            username='admin',
            password_hash=hash_password('admin123'),
            display_name='管理员',
            role='admin',
            team_id=team.id,
        )
        db.add(admin)
        await db.commit()
        print('管理员创建成功: admin / admin123')

asyncio.run(setup())
"
```

> 请在首次登录后立即修改默认密码。

## 7. 监控与日志

### 7.1 健康检查

```bash
# API 健康检查
curl http://localhost:8080/api/health
# 返回: {"status": "ok"}
```

### 7.2 日志位置

| 组件 | 日志位置 |
|------|----------|
| API 服务 | stdout / systemd journal / gunicorn access log |
| 引擎执行 | `--report-dir` 下的 JSON 报告 |
| Nginx | `/var/log/nginx/access.log` |

### 7.3 监控建议

- 使用 `/api/health` 端点配置存活探针（Kubernetes liveness probe）
- 监控 API 响应时间（Nginx access log 或 Prometheus）
- 监控磁盘使用（报告文件会持续增长）
- 定期清理过期报告文件

## 8. 安全加固

### 8.1 生产环境必做

- [ ] 修改默认管理员密码
- [ ] 设置 `SECRET_KEY` 环境变量（不使用默认值）
- [ ] 配置 HTTPS（Let's Encrypt 或企业证书）
- [ ] 限制 `CORS_ORIGINS` 为实际前端域名
- [ ] 配置防火墙，仅开放 80/443 端口
- [ ] 数据库文件权限设为 600

### 8.2 可选加固

- 启用 rate limiting（Nginx `limit_req`）
- 配置 WAF（如 ModSecurity）
- 启用审计日志
- 配置自动备份（cron + rsync/S3）

## 10. 常见问题

### Q: 启动报错 "address already in use"

```bash
# 查找占用端口的进程
lsof -i :8080
# 或
netstat -tlnp | grep 8080
# 终止进程后重启
```

### Q: SQLite 并发写入报错

SQLite 不支持高并发写入。如果遇到 `database is locked` 错误：
- 减少 gunicorn worker 数量（建议 2-4）
- 或迁移到 PostgreSQL

### Q: 前端页面空白

确认 Nginx 配置了 `try_files $uri $uri/ /index.html`，Vue Router 的 history 模式需要此配置。

### Q: 压测任务超时

调整 gunicorn 的 `--timeout` 参数（默认 120s），或在 Nginx 中增加 `proxy_read_timeout`。
