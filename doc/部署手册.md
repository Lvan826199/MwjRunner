## 1. 环境要求

| 组件 | 最低要求 | 说明 |

| 操作系统 | Linux (Ubuntu 20.04+ / CentOS 8+) / Windows Server 2019+ / macOS 12+ | 推荐 Ubuntu 22.04 |

## 2. 引擎部署（MwjRunner CLI）

引擎是核心组件，可以独立使用，不需要 Web 平台。

### 2.1 安装

```bash
# 从 Gitee 克隆项目
git clone https://gitee.com/xiaozai-van-liu/MwjRunner.git
cd MwjRunner

# 安装 Python 依赖（使用 UV）
uv sync

# 验证安装成功
uv run mwjrunner --help
```

如果看到命令帮助信息，说明安装成功。

### 2.2 配置环境文件

```bash
# 创建环境配置目录
mkdir -p envs

# 创建生产环境配置
cat > envs/prod.yaml << 'EOF'
name: production
base_url: https://api.your-domain.com
auth:
  type: bearer
  token: ${PROD_API_TOKEN}
variables:
  timeout: 30
EOF
```

### 2.3 执行用例

```bash
# 单文件执行
uv run mwjrunner run cases/smoke.yaml --env prod --report json,html

# 目录批量执行
uv run mwjrunner run cases/ --env prod --tags smoke --workers 4 --report json,html

# 指定报告输出目录
uv run mwjrunner run cases/ --env prod --report-dir /data/reports
```

### 2.4 定时任务（Linux cron）

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 2 点执行回归测试
0 2 * * * cd /opt/MwjRunner && uv run mwjrunner run cases/ --env prod --tags regression --report json,html --report-dir /data/reports 2>&1 >> /var/log/mwjrunner.log
```

## 3. 平台部署（Web 管理界面）

### 3.1 后端部署

```bash
cd platform/backend

# 安装依赖
uv sync

# 启动（开发模式）
uv run uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

# 启动（生产模式，使用 gunicorn + uvicorn worker）
uv run gunicorn app.main:app \
  -w 4 \
  -k uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8080 \
  --timeout 120 \
  --access-logfile /var/log/mwjrunner-api.log
```

**环境变量：**

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `DATABASE_URL` | 数据库连接串 | `sqlite+aiosqlite:///data/mwjrunner.db` |
| `CORS_ORIGINS` | 允许的跨域来源 | `*` |
| `SECRET_KEY` | JWT 签名密钥（生产必须修改） | 随机生成 |
| `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` | Access Token 有效期（分钟） | `30` |
| `JWT_REFRESH_TOKEN_EXPIRE_DAYS` | Refresh Token 有效期（天） | `7` |

### 3.2 前端构建与部署

```bash
cd platform/frontend

# 安装依赖
npm install

# 构建生产包
npm run build

# 产物在 dist/ 目录，部署到 Nginx
```

### 3.3 Nginx 配置

```nginx
server {
    listen 80;
    server_name mwjrunner.your-domain.com;

    # 前端静态文件
    location / {
        root /opt/MwjRunner/platform/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400s;
    }
}
```

### 3.4 Systemd 服务（Linux）

```ini
# /etc/systemd/system/mwjrunner-api.service
[Unit]
Description=MwjRunner API Server
After=network.target

[Service]
Type=simple
User=mwjrunner
WorkingDirectory=/opt/MwjRunner/platform/backend
ExecStart=/opt/MwjRunner/platform/backend/.venv/bin/gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8080 --timeout 120
Restart=always
RestartSec=5
Environment=DATABASE_URL=sqlite+aiosqlite:///opt/MwjRunner/platform/backend/data/mwjrunner.db
Environment=SECRET_KEY=your-production-secret-key-change-me

[Install]
WantedBy=multi-user.target
```

```bash
# 启用并启动服务
sudo systemctl daemon-reload
sudo systemctl enable mwjrunner-api
sudo systemctl start mwjrunner-api
sudo systemctl status mwjrunner-api
```

## 4. Docker 部署（推荐）

### 4.1 Dockerfile（后端）

```dockerfile
FROM python:3.13-slim

WORKDIR /app

# 安装 UV
RUN pip install uv

# 复制项目
COPY platform/backend/ .

# 安装依赖
RUN uv sync --frozen

# 创建数据目录
RUN mkdir -p data storage

EXPOSE 8080

CMD ["uv", "run", "uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8080"]
```

### 4.2 Dockerfile（前端）

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY platform/frontend/package*.json ./
RUN npm ci
COPY platform/frontend/ .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY deploy/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### 4.3 Docker Compose

```yaml
version: '3.8'

services:
  api:
    build:
      context: .
      dockerfile: deploy/Dockerfile.backend
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data
      - ./storage:/app/storage
    environment:
      - DATABASE_URL=sqlite+aiosqlite:///app/data/mwjrunner.db
      - SECRET_KEY=your-production-secret-key-change-me
    restart: always

  web:
    build:
      context: .
      dockerfile: deploy/Dockerfile.frontend
    ports:
      - "80:80"
    depends_on:
      - api
    restart: always
```

```bash
# 一键启动
docker-compose up -d

# 查看日志
docker-compose logs -f api

# 停止
docker-compose down
```

## 5. 数据库迁移

当前使用 SQLAlchemy 自动建表（`Base.metadata.create_all`），首次启动自动创建所有表。

如需手动迁移或升级：

```bash
cd platform/backend

# 查看当前表结构
uv run python -c "
from app.models import Base
for table in Base.metadata.sorted_tables:
    print(f'{table.name}: {[c.name for c in table.columns]}')
"
```

**生产环境建议：**
- 定期备份 SQLite 数据库文件（`data/mwjrunner.db`）
- 大规模使用时迁移到 PostgreSQL
- 使用 Alembic 管理数据库版本迁移

## 6. 初始化管理员

首次部署后，创建管理员账号：

```bash
cd platform/backend

uv run python -c "
import asyncio
from app.core.database import async_session_factory, init_db
from app.models.user import User, Team
from app.core.security import hash_password

async def setup():
    await init_db()
    async with async_session_factory() as db:
        # 创建默认团队
        team = Team(name='默认团队', description='系统默认团队')
        db.add(team)
        await db.flush()

        # 创建管理员
        admin = User(
            username='admin',
            password_hash=hash_password('admin123'),
            display_name='管理员',
            role='admin',
            team_id=team.id,
        )
        db.add(admin)
        await db.commit()
        print('管理员创建成功: admin / admin123')

asyncio.run(setup())
"
```

> ⚠️ 请在首次登录后立即修改默认密码。

## 8. 安全加固

### 8.1 生产环境必做

- [ ] 修改默认管理员密码
- [ ] 设置 `SECRET_KEY` 环境变量（不使用默认值，建议 32 位以上随机字符串）
- [ ] 配置 HTTPS（Let's Encrypt 或企业证书）
- [ ] 限制 `CORS_ORIGINS` 为实际前端域名
- [ ] 配置防火墙，仅开放 80/443 端口
- [ ] 数据库文件权限设为 600

### 8.2 可选加固

- 启用 rate limiting（Nginx `limit_req`）
- 配置 WAF（如 ModSecurity）
- 启用审计日志
- 配置自动备份（cron + rsync/S3）

## 10. 常见问题

### Q: 启动报错 "address already in use"

```bash
# 查找占用端口的进程
lsof -i :8080
# 或
netstat -tlnp | grep 8080
# 终止进程后重启
```

### Q: SQLite 并发写入报错

SQLite 不支持高并发写入。如果遇到 `database is locked` 错误：
- 减少 gunicorn worker 数量（建议 2-4）
- 或迁移到 PostgreSQL

### Q: 前端页面空白

确认 Nginx 配置了 `try_files $uri $uri/ /index.html`，Vue Router 的 history 模式需要此配置。

### Q: 压测任务超时

调整 gunicorn 的 `--timeout` 参数（默认 120s），或在 Nginx 中增加 `proxy_read_timeout`。

### Q: WebSocket 连接失败

确认 Nginx 配置了 WebSocket 代理（见 3.3 节），需要 `proxy_http_version 1.1` 和 `Upgrade` 头。

### Q: Token 过期后无法自动刷新

检查前端是否正确配置了 axios 拦截器。正常情况下 401 响应会自动触发 refresh token 请求。如果 refresh token 也过期（7 天），需要重新登录。

### Q: UV 安装失败

```bash
# 如果 pip install uv 失败，可以用官方安装脚本
curl -LsSf https://astral.sh/uv/install.sh | sh
# 安装后重新打开终端或 source ~/.bashrc
```

## 12. Linux 从零部署全流程（小白版）

本节面向完全没有部署经验的用户，从一台全新的 Linux 服务器开始，一步步完成 MwjRunner 平台的完整部署。

### 前提条件

- 一台 Linux 服务器（推荐 Ubuntu 22.04，最低 2 核 4GB 内存）
- 能通过 SSH 登录服务器（有 root 或 sudo 权限）
- 服务器能访问外网（需要下载软件包）

### 第一步：连接服务器

```bash
# 在你的电脑上打开终端，通过 SSH 连接服务器
ssh root@你的服务器IP
# 输入密码后回车
```

### 第二步：安装系统依赖

```bash
# 更新系统包列表
apt update && apt upgrade -y

# 安装基础工具
apt install -y git curl wget build-essential libssl-dev zlib1g-dev \
  libbz2-dev libreadline-dev libsqlite3-dev libffi-dev lzma liblzma-dev

# 安装 Nginx（用于反向代理）
apt install -y nginx

# 启动 Nginx 并设为开机自启
systemctl enable nginx
systemctl start nginx
```

### 第三步：安装 Python 3.13

```bash
# 安装 pyenv（Python 版本管理器）
curl https://pyenv.run | bash

# 添加 pyenv 到环境变量（写入 ~/.bashrc）
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc

# 让配置生效
source ~/.bashrc

# 安装 Python 3.13
pyenv install 3.13
pyenv global 3.13

# 验证
python --version
# 应该显示 Python 3.13.x
```

### 第四步：安装 UV（Python 包管理器）

```bash
# 安装 UV
curl -LsSf https://astral.sh/uv/install.sh | sh

# 让配置生效
source ~/.bashrc

# 验证
uv --version
```

### 第五步：安装 Node.js 18（前端构建用）

```bash
# 安装 nvm（Node 版本管理器）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# 让配置生效
source ~/.bashrc

# 安装 Node.js 18
nvm install 18
nvm use 18

# 验证
node --version
npm --version
```

### 第六步：拉取项目代码

```bash
# 创建项目目录
mkdir -p /opt
cd /opt

# 从 Gitee 克隆代码
git clone https://gitee.com/xiaozai-van-liu/MwjRunner.git
cd MwjRunner
```

### 第七步：部署引擎

```bash
# 在项目根目录安装引擎依赖
cd /opt/MwjRunner
uv sync

# 验证引擎可用
uv run mwjrunner --help

# 如果看到帮助信息，引擎部署成功！
```

### 第八步：部署后端 API

```bash

cd /opt/MwjRunner/platform/backend

# 安装后端依赖
uv sync

# 创建数据目录
```


### 第十步：配置 Nginx

```bash
# 创建 Nginx 配置文件
cat > /etc/nginx/sites-available/mwjrunner << 'EOF'
server {
    listen 80;
    server_name _;

    # 前端静态文件
    location / {
        root /opt/MwjRunner/platform/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400s;
    }
}
EOF

# 启用配置（创建软链接）
ln -sf /etc/nginx/sites-available/mwjrunner /etc/nginx/sites-enabled/mwjrunner

# 删除默认配置（避免冲突）
rm -f /etc/nginx/sites-enabled/default

# 测试配置是否正确
nginx -t

# 重新加载 Nginx
systemctl reload nginx
```

```bash
# 查看后端日志
journalctl -u mwjrunner-api -f

# 重启后端
systemctl restart mwjrunner-api

# 重启 Nginx
systemctl restart nginx

# 更新代码
cd /opt/MwjRunner
git pull origin master
cd platform/backend && uv sync
cd ../frontend && npm install && npm run build
systemctl restart mwjrunner-api
systemctl reload nginx
```

### 开放防火墙端口

```bash
# 如果使用 ufw（Ubuntu 默认防火墙）
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 22/tcp
ufw enable

# 如果使用 firewalld（CentOS）
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --reload
```

### CentOS / RHEL 用户注意

如果你的服务器是 CentOS 8+ 或 RHEL 8+，第二步的命令需要替换：

```bash
# CentOS/RHEL 使用 dnf 代替 apt
dnf update -y
dnf groupinstall -y "Development Tools"
dnf install -y git curl wget openssl-devel zlib-devel bzip2-devel \
  readline-devel sqlite-devel libffi-devel xz-devel nginx

systemctl enable nginx
systemctl start nginx
```

Nginx 配置文件路径也不同：
- 配置文件放在 `/etc/nginx/conf.d/mwjrunner.conf`（不需要 sites-available/sites-enabled）

## 13. 生成 SECRET_KEY

生产环境必须使用随机密钥，不要使用默认值：

```bash
# 方法 1：使用 Python 生成
python -c "import secrets; print(secrets.token_urlsafe(32))"

# 方法 2：使用 openssl 生成
openssl rand -base64 32
```

将生成的字符串填入 systemd 服务文件的 `SECRET_KEY` 环境变量中，然后重启服务：

```bash
# 编辑服务文件
nano /etc/systemd/system/mwjrunner-api.service

# 修改 SECRET_KEY 那一行后保存，然后：
systemctl daemon-reload
systemctl restart mwjrunner-api
```
